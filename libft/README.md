# Libft

## Description

Libft is a custom C library that reimplements a collection of standard C library functions along with additional utility functions. The library serves as a foundational toolset for future 42 projects, covering string manipulation, memory operations, character classification, linked list management, and file descriptor output.

The library is split into three parts:

**Part 1 — Libc functions:** Reimplementations of standard C library functions such as `memset`, `strlen`, `strchr`, `strncmp`, `atoi`, `calloc`, `strdup`, and more. These follow the same prototypes and behavior as their originals, prefixed with `ft_`.

**Part 2 — Additional functions:** Utility functions not found in the standard library, including `ft_substr`, `ft_strjoin`, `ft_strtrim`, `ft_split`, `ft_itoa`, `ft_strmapi`, `ft_striteri`, and output functions like `ft_putchar_fd`, `ft_putstr_fd`, `ft_putendl_fd`, and `ft_putnbr_fd`.

**Part 3 — Linked list functions:** A set of functions for creating and manipulating singly linked lists, including `ft_lstnew`, `ft_lstadd_front`, `ft_lstadd_back`, `ft_lstsize`, `ft_lstlast`, `ft_lstdelone`, `ft_lstclear`, `ft_lstiter`, and `ft_lstmap`.

## Instructions

### Compilation

Clone the repository and compile the library using `make`:

```
git clone <repository-url> libft
cd libft
make
```

This produces `libft.a`, a static library archive. The following Makefile rules are available:

- `make` or `make all` — compiles the library
- `make clean` — removes object files
- `make fclean` — removes object files and the library archive
- `make re` — performs a full recompilation

### Usage

To use libft in your project, include the header and link the library:

```c
#include "libft.h"
```

Compile with:

```
cc your_file.c -L/path/to/libft -lft -I/path/to/libft
```

### AI Usage

AI (Claude by Anthropic) was used as a learning companion throughout this project. It was not used to generate solutions directly. Specifically:

- **Code review and debugging:** Every function was written independently first, then submitted to AI for review. The AI helped identify bugs such as undefined behavior in expressions, unsigned integer overflow edge cases, and operator precedence issues with type casts.
- **Concept explanations:** AI was used to understand C concepts including the difference between `void *` and typed pointers, why `unsigned char` is used for raw memory operations, how `const` works at the compiler level, overlapping memory handling in `memmove`, and linked list node management.
- **Testing support:** AI helped write test `main` functions and identified important edge cases to verify, such as `NULL` inputs, empty strings, zero-size allocations, and integer overflow in `calloc`.
- **Man page clarification:** When terminal man pages were unclear or unavailable (such as `strlcat` and `strnstr`), AI was used to explain function behavior, return values, and edge cases.

AI was also used to generate the initial draft of this README.md which was reviewed and approved before submission.
No function implementation was generated by AI. Every solution originated from independent reasoning and was iteratively improved through review and discussion.
